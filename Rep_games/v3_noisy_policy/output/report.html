V3：噪声历史中的行为与语言线索

V3 crosses behavior pattern and language frame. Every observed history has average return 0.40, but individual return fractions are noisy. This tests whether language changes trust or prediction after behavior is controlled.

Successful calls: 72 / 72.

实验 protocol 和操作细节

V3 在 V2 的基础上加入噪声，并正交组合行为模式与语言框架，检验在不那么整齐的历史中，温暖承诺或道歉解释是否会改变模型判断。

方法总览

4 behavior patterns × 3 language frames × 6 repetitions = 72 runs；成功解析 72/72。所有行为模式的研究者计算平均返还都保持在约 0.40。

任务规则和信息呈现

项目具体操作
投资规则low / medium / high stake 最大投资分别为 4、7、10；投资乘 3 给 partner。
历史呈现每轮呈现 stake、partner message、投资 token、partner received、partner returned。模型看 token 数，不直接看分析用 return_fraction。
噪声设置每个 behavior pattern 的返还围绕目标结构波动，不再是 V2 那样整齐的固定序列。
语言操纵同一类行为结构与三类语言框架交叉组合，使语言和行为在设计上分离。

关键变量和计算方式

每个 run / trial 的完整流程

  1. 脚本从 conditions/factorial_design.json 读取 4 个行为模式和 3 个语言框架。
  2. 每个 cell 生成 6 个 noisy histories；每段历史包含 6 轮。
  3. 模型一次性阅读完整历史后，输出信任、三种 stake 下的预测和投资。
  4. 额外要求模型自报 message_weight 和 behavior_weight，用于诊断它自认为的信息来源。
  5. 分析行为主效应、语言主效应，以及 high-low prediction gap。

模型输出字段

JSON schema 包括 trust_rating, 三个返还预测、三个投资计划、message_weight, behavior_weight, confidence, brief_reason。

需要注意的限制

  • V3 的 message_weight / behavior_weight 会显式提示模型比较语言和行为，可能污染信息使用结构；V4 因此去掉这类探针。

主要结果

V3 把行为模式和语言框架正交组合，并加入噪声。结果延续 V2：行为结构仍然主导信任、条件预测和投资；语言有小幅影响，但不稳定，也没有覆盖行为证据。

20.1自报语言权重

-0.299机会主义者 high-low gap

核心解释：加噪声后，模型仍能识别 partner policy。语言框架对 trust 有一些波动，但不是主要驱动。

结论与下一步

V3 说明，在更有噪声的历史中，模型依然主要看实际行为结构，而不是简单被 warm promise 或 apology excuse 牵引。但 V3 直接询问了 message_weight 和 behavior_weight，这会污染任务结构。V4 因此去掉显性权重问题，改用 WTP 和下一轮投资这样的 costly choice。
