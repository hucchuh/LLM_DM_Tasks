V7 收益匹配后的诚实迁移实验

核心问题：当建议带来的实际收益被严格匹配之后，模型是否还会因为一个对象更诚实，而更愿意付费、进入互动、投入更多资源？

实验 protocol 和操作细节

V7 在 V6 基础上匹配 realized advice payoff：诚实组和不诚实组的 recommendation win rate 都是 0.50，只改变 factual honesty，测试 honesty 是否仍会迁移到后续 costly reliance。

方法总览

2 adviser types × 2 presentation modes × 16 seeds = 64 runs；成功解析 64/64。每个 run 先经历 12 轮 observation episodes，再进行一次主投资决策；honesty/helpfulness 只在独立 probe 中测量。

任务规则和信息呈现

项目具体操作
observation task模型观察 advice episodes，但在 sequential_observation 中不选择卡牌，只确认观察到当前轮。

关键变量和计算方式

变量类型/范围含义

moral_trust独立 probe，0-100更偏道德评价的信任判断。

每个 run / trial 的完整流程

  1. 脚本生成 high/low honesty 两组 partner，各自 12 轮 advice episodes，确保两组推荐胜率相同。
  2. sequential_observation 中，模型逐轮看到 partner message 和反馈，但每轮只返回当前轮编号，避免在观察阶段引入 trust probe。
  3. batch_review 中，模型一次性阅读同样 12 轮历史。
  4. 主决策阶段，模型只输出 enter_choice, willingness_to_pay, investment。
  5. 随后用独立 probe prompt 测 perceived honesty、helpfulness、expected win rate 和 moral trust。

模型输出字段

主决策 JSON: enter_choice, willingness_to_pay, investment。Probe JSON: perceived_honesty, perceived_helpfulness, expected_recommendation_win_rate, moral_trust。

需要注意的限制

  • V7 的主投资仍发生在完整历史之后，不是真正逐轮投资行为；V8 因此改成 trial-by-trial investment。

为什么做 V7

V6 已经证明模型可以把“这个人说真话”和“这个人让我赚钱”分开。但那个设计里 honesty 和 payoff 是两个很清楚的变量，容易被模型当成一道分类题。V7 因此把两类对象的建议收益固定为完全一样：12 轮里都只有 6 次建议会赢。唯一改变的是事实陈述是否诚实。

这样可以问一个更干净的问题：如果一个人并没有让模型赚更多钱，只是更诚实，模型会不会把这种诚实迁移到后续的付费和投资行为上？

实验设计

每个 run 先看到同一个 partner 的 12 轮信息建议记录。每轮包含：partner 声称自己看到某张牌的数值、给出左右选择建议，随后反馈真实牌面、陈述是否真实、建议是否会赢。

两个条件的建议收益完全相同，都是 6/12 次建议会赢；区别只在事实陈述：诚实条件为 9/12 次真实，不诚实条件为 3/12 次真实。之后模型进入一次新的投资互动，需要输出是否进入、最多愿意支付多少入场费、以及投入多少 token。honesty 和 helpfulness 只在之后的 probe 中测量，不在主决策里询问。

总体上，诚实对象获得更高的道德信任，也获得更高的 WTP 和投资额。这说明在收益完全匹配时，honesty 并没有被模型视为纯粹无关的道德标签，而是会部分迁移到后续的 costly reliance。

但这个迁移并不稳定地出现在所有呈现方式中。逐轮观察时，诚实对象的 WTP 只高 0.687，投资额反而低 0.5；一次性回看时，诚实对象的 WTP 高 1.5，投资额高 3.126。这提示“信息如何进入上下文”本身会改变模型把社会评价转化为行动的方式。

图示结果

这个拆分可能是 V7 最值得继续追的地方。batch 更像在做总结判断，honesty 更容易进入最终决策；sequential 更像真实互动，模型持续接收反馈，但最后付费和投资没有完全跟随 moral trust。这给下一版留下了一个清楚问题：模型是如何在连续互动中把“我认为你诚实”和“我愿意承担风险”连接或分离的？
